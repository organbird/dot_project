| 작성일 | 2026-02-07 |

---

## 1. 액터 정의

| 액터 | 설명 |
|------|------|
| 비회원 (Guest) | 시스템에 로그인하지 않은 사용자 |
| 일반 사용자 (User) | 시스템에 로그인한 일반 사용자 |
| 관리자 (Admin) | 시스템 관리 권한을 가진 사용자 |
| AI 시스템 | 챗봇, 이미지 생성, 음성 변환을 수행하는 AI 서비스 |
| System Manager | AIDot Admin Portal을 통해 시스템을 배포·관리하는 운영 담당자 |

## 3. 유스케이스 명세

### 3.1 인증 시스템

| 유스케이스ID | 유스케이스명 | 개요 | 주요 액터 | 이해관계자/관심사항 | 선행조건 | 이벤트 흐름 | 고려사항 |
|-------------|-------------|------|----------|-------------------|----------|------------|----------|
| UC-AUTH-001 | 회원가입 | 신규 사용자가 시스템에 계정을 생성한다 | 비회원 (Guest) | ❍ 사용자: 간편한 가입 절차<br>❍ 관리자: 유효한 사용자 정보 확보<br>❍ 시스템: 데이터 무결성 보장 | ❍ 시스템 접속 상태<br>❍ 비로그인 상태 | ❍ 회원가입 페이지 접근<br>❍ 필수 정보 입력 (이메일, 이름, 비밀번호, 연락처, 성별)<br>❍ 부서 선택<br>❍ 가입 완료 버튼 클릭<br>❍ 입력값 검증 수행<br>❍ 계정 생성 완료<br>❍ 로그인 페이지 이동 | ❍ 이메일 중복 검사 필수<br>❍ 비밀번호 최소 4자 이상<br>❍ bcrypt 암호화 적용 |
| UC-AUTH-002 | 로그인 | 등록된 사용자가 시스템에 인증하여 접속한다 | 비회원 (Guest) | ❍ 사용자: 빠르고 안전한 로그인<br>❍ 시스템: 보안 인증 처리 | ❍ 등록된 계정 존재<br>❍ 비로그인 상태 | ❍ 로그인 페이지 접근<br>❍ 이메일, 비밀번호 입력<br>❍ 로그인 버튼 클릭<br>❍ bcrypt 비밀번호 검증<br>❍ JWT 토큰 발급<br>❍ 역할별 페이지 이동 | ❍ JWT 토큰 기반 인증<br>❍ USER→/home, ADMIN→/dashboard<br>❍ 로그인 실패 시 오류 메시지 |
| UC-AUTH-003 | 로그아웃 | 로그인된 사용자가 시스템에서 로그아웃한다 | 사용자 (User), 관리자 (Admin) | ❍ 사용자: 안전한 세션 종료<br>❍ 시스템: 보안 상태 유지 | ❍ 로그인 상태 | ❍ 사이드바 로그아웃 버튼 클릭<br>❍ localStorage 토큰 삭제<br>❍ 사용자 정보 삭제<br>❍ 메인 페이지로 이동 | ❍ 모든 인증 정보 완전 삭제<br>❍ 세션 즉시 무효화 |
