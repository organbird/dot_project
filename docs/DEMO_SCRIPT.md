## STEP 5. AI 챗봇 시연 (핵심 기능)

> **[화면: /chatbot]**

### 보여줄 화면
- 3단 레이아웃 (사이드바 | 세션 목록 | 채팅 영역)
- 실시간 스트리밍 응답
- RAG 기반 문서 참조 응답

### 말할 내용

> "AI 챗봇 페이지입니다. 왼쪽에 대화 세션 목록이 있고, 오른쪽이 실제 채팅 영역입니다."
>
> "먼저 새로운 대화를 만들어 보겠습니다."
>
> *(새 대화 생성 후)*
>
> "질문을 입력하면 **LLaMA 3 Bllossom 한국어 모델**이 실시간으로 응답을 생성합니다. 응답이 **스트리밍 방식**으로 토큰 단위로 표시되는 것을 확인할 수 있습니다."
>
> *(스트리밍 응답 표시 중)*
>
> "이렇게 한 글자씩 실시간으로 응답이 나오는 것이 스트리밍 방식입니다. 사용자가 기다리는 동안에도 응답을 바로 확인할 수 있어 UX가 크게 향상됩니다."
>
> *(RAG 시연 - 문서가 이미 업로드된 상태라면)*
>
> "업로드된 문서 내용을 기반으로 질문하면, AI가 해당 문서를 참고하여 답변합니다. 응답 하단에 **참고 문서** 정보가 표시됩니다. 이것이 RAG(Retrieval-Augmented Generation) 기술입니다."

### 조작
1. **"+ 새 대화"** 버튼으로 세션 생성
2. 간단한 질문 입력 (예: "프로젝트 관리 방법에 대해 알려주세요")
3. **스트리밍 응답이 실시간으로 표시되는 것을 보여주기** ← 핵심
4. 응답 중 **"중단"** 버튼 기능 보여주기 (선택)
5. (RAG 시연) 문서 관련 질문 → 참조 문서 표시 확인
6. 세션 제목 변경 기능 보여주기
7. 사이드바에서 **"이미지"** 클릭

## STEP 6. 문서 보관함 시연

> **[화면: /documents]**

### 보여줄 화면
- 문서 목록 테이블
- 카테고리 필터
- 문서 업로드 모달
- 문서 상세/다운로드

### 말할 내용

> "문서 보관함입니다. 업로드된 문서를 카테고리별로 관리할 수 있습니다."
>
> "문서를 업로드하면 **백그라운드에서 자동으로 RAG 벡터화가 진행**됩니다. 즉, 문서를 업로드하기만 하면 챗봇이 해당 문서를 참조할 수 있게 됩니다."

### 조작
1. 문서 목록 확인 (제목, 형식, 크기, 등록일)
2. **카테고리 필터** 클릭하여 필터링 시연
3. **검색** 기능 시연
4. **"파일 업로드"** 버튼 → 업로드 모달 보여주기
5. PDF 파일 업로드 시연 (미리 준비한 파일)
6. 문서 상세 보기 → 다운로드 기능 보여주기
7. 사이드바에서 **"이미지"** 클릭

>

>
> *(이미지 생성 요청 후)*
>
> "한글 프롬프트를 입력하면 **LLM이 자동으로 영어로 번역**한 뒤, Worker 서버의 ComfyUI에서 이미지를 생성합니다. Celery를 통한 비동기 처리로, 생성 중에도 다른 작업을 계속할 수 있습니다."
>
> *(이미지 생성 완료 후)*
>

1. 프롬프트 입력 (예: "아름다운 한국의 산과 호수 풍경")
2. 스타일 선택 (사실적)
3. **"생성하기"** 클릭
4. 생성 중 로딩 상태 보여주기
5. (미리 생성해둔 이미지가 있다면) 갤러리에서 이미지 확인
6. 이미지 클릭 → 상세 모달 보여주기
7. 사이드바에서 **"회의록"** 클릭

3. 업로드 후 상태 변화 보여주기
4. 완료된 회의록 클릭 → 상세 모달에서 전문/요약 확인
5. (또는) **"직접 작성"** 모달 보여주기

> "중간에는 **일별 활동 추이 차트**와 **AI 기능 사용 현황 차트**, **부서별 인원/활동 분포 차트**가 있습니다. 모두 **Recharts** 라이브러리로 구현했습니다."

> "하단에는 **실시간 시스템 로그**가 표시됩니다. 로그인, 문서 업로드, AI 채팅 등 모든 주요 활동이 기록됩니다."

### 조작
1. 관리자 계정으로 로그인 (또는 탭 전환)
2. 서버 리소스 바 (CPU/MEM/DISK) 확인
3. 통계 카드 4개 확인
4. 일별 활동 차트 확인
5. 기능별 사용 차트 확인
6. 부서별 파이 차트 / 막대 차트 확인
7. 시스템 로그 테이블 확인
8. 사이드바 **"사용자 관리"** 클릭

### 사용자 관리 (/admin/settings)

> "사용자 관리에서는 전체 사용자 목록을 조회하고, **권한을 변경**하거나 **비밀번호를 초기화**할 수 있습니다."

### 조작
1. 사용자 목록 확인
2. 사용자 선택 → 상세 정보 확인
3. 권한 변경 (USER ↔ ADMIN) 보여주기
4. 사이드바 **"부서 관리"** 클릭

### 부서 관리 (/admin/depts)

> "부서 관리에서는 부서를 추가/삭제하고, 소속 직원을 다른 부서로 이동할 수 있습니다."

### 조작
1. 부서 목록 확인
2. 부서 클릭 → 소속 직원 목록 확인
3. 부서 이동 기능 보여주기

## 비상 대응 가이드

### 시연 중 문제 발생 시

| 상황 | 대처 |
|------|------|
| AI 챗봇 응답이 느릴 때 | "로컬 LLM이라 첫 응답에 약간의 시간이 걸립니다" 후 다른 기능 먼저 시연 |
| 이미지 생성이 오래 걸릴 때 | "비동기 처리이므로 백그라운드에서 생성 중입니다" 후 미리 생성한 이미지 보여주기 |
| STT 변환이 오래 걸릴 때 | "Worker 서버에서 처리 중입니다" 후 이미 완료된 회의록 보여주기 |
| 서버 연결 오류 | Docker 컨테이너 상태 확인, 재시작 |
| 로그인 실패 | 다른 테스트 계정 사용 |

### 미리 준비해둘 데이터

1. **AI 대화 세션** 2-3개 (질문-응답이 있는 상태)
2. **업로드된 문서** 3-5개 (PDF, DOC 등 다양한 형식)
3. **생성된 이미지** 4-5개 (다양한 스타일)
4. **완료된 회의록** 1-2개 (전문 + 요약이 있는 상태)
5. **등록된 일정** 3-5개 (다양한 카테고리)
